Amazon EC2 Security Group 设置

  1. 若留空，则什么都不允许通过。

  2. 要允许所有的流量通过，可以设置All traffic 中的 source为anywhere。

  3. 若是搭建了Shadowsocks服务，Security Group没有设置好导致sock流量被阻拦，浏览器中看上去是和被墙的情况一样的，即是connection reset。

  4. source 设置为anywhere会包括ipv6,可以删除。

  5. 设置Inbound规则为允许Shadowsocks对外端口开放，选择Custom TCP Rule，设置相应端口，source为anywhere。

    如: Custom TCP TCP 8080 0.0.0.0/0(也可以设置为MY ip私用)

  6. Outbound中如果设置了Shadowsocks特定端口，并不能使流量顺利通过，通过netstat查看的确连接的是云端的9999端口啊！？ 原因不明（目前仅是要设置成全部tcp流量通过才可）。

  7. 设置固定ip连接时，如果自身ip有变动，页面显示也是页面被重置的状态。
