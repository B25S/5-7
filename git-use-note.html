    git使用笔记


    git是一个分布式版本控制系统，起初由linus开发的用来对linux核心代码的管理工具。


    git push test test1:remotetest

    push之后会删除远程的remotetest分支，push上去的test会代替，但是可以通过回滚恢复。


    git push origin 本地分支名称:远程分支名称

    若本地分支名称与远程分支名称相同，则为更新远程分支，如果不同，则会创建一个新的远程分支并且更新。


    git pull 远程库名称 远程分支名称

    将远程仓库的分支抓取下来，获取最新的版本,会指向最新的head


    git log

    这个命令可以查看日志，其中会记录最近commit的hash值，配合reset命令可以回滚到指定的commit版本


    git reset --hard HEAD^

    HEAD代表当前commit版本的指针，改变版本号其实就是移动这个指针。

    HEAD^ 代表上一个，也可以写成HEAD~x，如回退第前十个版本为: HEAD~10;当然用git log中显示的hash值也是可以的:git reset --hard hash。


    git比较命令

    git diff 版本1 版本2 -- 文件路径

    git show 在git pull之后会看到最新的commit与前一个commit的不同，会显示文件的具体内容。
